A wrapper class which interacts with the git repositories.

class GitPublisher(data, package, handler=None)[source]

Bases: rez_docbot.bases.base.Publisher

The wrapper class which interacts with the git repositories.

It also handles cloning and pushing to remote repositories, like GitHub.

Important

This class is a boilerplate for other classes to subclass and extend. It is not meant to be used on its own.

get_resolved_view_url()[source]

Create a viewable URL where documentation can be seen.

This differs from the publish URL. For example, the publish URL for GitHub is typically "git@github.com:UserName/{package.name}" but the documentation is viewed in "https://UserName.github.io/{package.name}".

quick_publish(documentation)[source]

Clone, copy, and push documentation as required.

Parameters

documentation (str) – The absolute directory to built documentation on-disk.

Raises

  • RuntimeError – If this instance hasn’t authenticated to a VCS remote, e.g. GitHub.

  • MissingDocumentation – If documentation was expected to be added for commit + push but, for some unknown reason, the git repository cannot commit + push it. (This usually happens because of a messed up .gitignore file or .gitignore_global).

  • CannotMakeDocumentation – If this instance disables “latest” folder publishes but the given Rez package has no version, then there’s literally nothing that can be published. If this occurs, it’s to be treated as user / administrator error (not rez_docbot’s fault).
